## APERCU DU PROJET
SmartPi-armbian est un constructeur d'images personnalisees pour les appareils SmartPi
basé sur le systeme d'exploitation Armbian. Le projet fournit des processus automatises
de build et de conversion pour creer des images Linux adaptees au materiel SmartPi.

Repository: https://github.com/Yumi-Lab/SmartPi-armbian
License: GPL-3.0

## PROCESSUS DE BUILD

### 1. Creation de la matrice (setup job)
- Decouverte des fichiers *.conf dans configs/
- Exclusion de config-default.conf
- Generation timestamp UTC

### 2. Build parallele (build job)
- Checkout du repo
- Build via actions/build-image
- Renommage: {VENDOR}-{board}-{codename}-{distro_version}-{variant}-{timestamp}.img
- Compression xz
- Calcul SHA256
- Upload artifacts

### 3. Conversion DietPi (dietpi job) - Server uniquement
- Download artifact Armbian
- Montage image en loop device
- Chroot avec QEMU ARM
- Installation DietPi
- Renommage: {board}-DietPi-{codename}-debian{version}-{variant}-{timestamp}.img
- Recompression et nouveau checksum

## CONVENTION DE NOMMAGE DES IMAGES

### Images Armbian
Format: `{VENDOR}-{board}-{codename}-{distro_version}-{variant}-{timestamp}.img.xz`
Exemple: `Yumi-smartpi1-bookworm-debian12-server-2026-02-02-1234.img.xz`

### Images DietPi
Format: `{board}-DietPi-{codename}-debian{version}-{variant}-{timestamp}.img.xz`
Exemple: `smartpi1-DietPi-bookworm-debian12-server-2026-02-02-1234.img.xz`

## SYSTEME DE CONFIGURATION FIRST-BOOT

La partition /boot est en FAT, accessible depuis Windows/Mac/Linux.
Le systeme est 100% compatible Raspberry Pi Imager.

### Methode 1: Fichier SmartPi natif
Fichier: /boot/smartpi-config.txt
Parametres disponibles:
- APPLY_CONFIG=1      # Activer la configuration (OBLIGATOIRE)
- HOSTNAME=xxx        # Nom de la machine
- SSH_ENABLED=1       # Activer SSH
- USERNAME=xxx        # Creer un utilisateur
- USER_PASSWORD=xxx   # Mot de passe utilisateur
- ROOT_PASSWORD=xxx   # Mot de passe root
- WIFI_SSID=xxx       # SSID WiFi
- WIFI_PASSWORD=xxx   # Mot de passe WiFi
- WIFI_COUNTRY=FR     # Code pays WiFi
- STATIC_IP=x.x.x.x   # IP statique
- NETMASK=x.x.x.x     # Masque reseau
- GATEWAY=x.x.x.x     # Passerelle
- DNS=x.x.x.x         # Serveur DNS
- TIMEZONE=xxx        # Fuseau horaire
- LOCALE=xxx          # Locale systeme

### Methode 2: Fichiers Raspberry Pi Imager (compatibilite)
Fichiers supportes sur /boot:
- ssh ou ssh.txt      # Fichier vide pour activer SSH
- wpa_supplicant.conf # Config WiFi standard
- userconf.txt        # Format: user:password_hash
- hostname            # Nom de machine en texte brut
- firstrun.sh         # Script custom execute au premier boot

### Service: smartpi-firstboot.service
- S'execute au premier demarrage
- Traite d'abord les fichiers Raspberry Pi Imager
- Puis applique smartpi-config.txt si APPLY_CONFIG=1
- Se desactive automatiquement apres execution
- Log: /var/log/smartpi-firstboot.log

## CHROOT DIETPI

### Problemes resolus:
1. DNS: Copie de /etc/resolv.conf
2. QEMU: Injection de qemu-arm-static
3. systemctl: Wrapper fake qui gere enable/disable/mask

### Variables d'environnement:
- HW_MODEL="25" (Allwinner H3 generic)
- DISTRO_TARGET: 6 (bullseye), 7 (bookworm), 8 (trixie)
- IMAGE_CREATOR="Yumi"
- PREIMAGE_INFO="Armbian"

## NOTES TECHNIQUES

### Emulation ARM
- QEMU user-static pour builds cross-architecture
- binfmt-support pour switch automatique
- Docker tonistiigi/binfmt pour setup complet

### Compression
- Format: xz
- Options: -efkvzT0 (max compression, multi-thread)

### Checksums
- Algorithme: SHA256
- Fichiers: .img.sha256 et .img.xz.sha256

## POUR MODIFIER LE PROJET

### Ajouter une nouvelle distribution:
1. Creer configs/{board}-{codename}-{variant}.conf
2. Ajouter le mapping codename->version dans BuildImages.yml
3. Si DietPi supporte: ajouter dans la matrice dietpi + mapping dans convert-dietpi

### Ajouter une nouvelle carte:
1. Creer boards/{board}.wip
2. Creer configs/{board}-{codename}-{variant}.conf pour chaque combo
3. Ajouter personnalisations dans customize-image.sh si necessaire

### Modifier la personnalisation:
- userpatches/customize-image.sh pour scripts
- userpatches/overlay/ pour fichiers a copier
