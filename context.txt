# SmartPi Armbian - État du projet (2026-02-04)

## Résumé de la session

### Problème initial
Les workflows GitHub Actions ne se déclenchaient plus automatiquement après commit, et l'image DietPi générée avait des erreurs au boot :
- `/boot/dietpi/.hw_model: line 1: 25: command not found`
- `((: == 75 : syntax error: operand expected`

### Solutions implémentées

#### 1. Correction du fichier .hw_model
**Problème** : Le fichier `.hw_model` contenait juste "25" alors que DietPi le source comme un script bash.

**Solution** : Créer `.hw_model` comme script bash avec toutes les variables requises :
```bash
G_HW_MODEL=25
G_HW_MODEL_NAME='Generic Allwinner H3'
G_HW_ARCH=2
G_HW_ARCH_NAME='armv7l'
G_HW_CPUID=0
G_HW_CPU_CORES=4
G_DISTRO=7
G_DISTRO_NAME='bookworm'
G_ROOTFS_DEV='/dev/mmcblk0p1'
G_HW_UUID=''
```

**Fichiers modifiés** :
- `actions/prepare-dietpi/action.yml`
- `actions/convert-dietpi/action.yml`

#### 2. Deux approches DietPi en parallèle

##### A. dietpi-qemu (QEMU/VM)
- Conversion complète via QEMU dans GitHub VM
- Exécute `dietpi-installer` en chroot avec émulation ARM
- `.install_stage=-1` (first-run setup à faire)
- Image prête à l'emploi mais nécessite QEMU

##### B. dietpi-firstboot (100% Offline)
- Clone le repo DietPi complet
- Installe TOUS les scripts dans `/boot/dietpi/` et `/usr/local/sbin/`
- `.install_stage=2` (installation complète, pas de wizard)
- Services systemd : `dietpi-boot`, `dietpi-postboot`, `dietpi-ramdisk`
- **Aucun internet requis au boot**
- Mot de passe root : `dietpi`

#### 3. Workflow simplifié (temporaire)

**Configuration actuelle** :
- Build : `smartpi1-bookworm-server` uniquement (Debian 12)
- DietPi QEMU : `smartpi1-bookworm-server` (distro_target=7)
- DietPi Offline : `smartpi1-bookworm-server` (distro_target=7)

**Raison** : DietPi v10+ ne supporte plus Debian 11 (Bullseye), minimum Debian 12 requis.

## Notes techniques

### prepare-dietpi (Offline)
- Clone `https://github.com/MichaIng/DietPi.git` (depth=1)
- Copie TOUT le repo dans l'image
- Pas d'exécution de dietpi-installer
- Services systemd custom pour boot/postboot/ramdisk
- .hw_model créé avec variables bash complètes

### convert-dietpi (QEMU)
- Télécharge dietpi-installer
- Exécute dans chroot avec QEMU ARM emulation
- Fake systemctl wrapper pour chroot
- Force .hw_model (format bash) et .install_stage après installation
